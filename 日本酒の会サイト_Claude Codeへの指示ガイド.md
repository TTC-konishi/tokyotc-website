# 日本酒の会サイト — Claude Code への指示ガイド

*小西さん用の備忘。日本酒の会サイト（`tokyotc.com/sake/` など）の作業を Claude Code に頼むときの「言い方」メモ。困ったらこれを見る。*

---

## 1. まず押さえる基本

- **`main`（メイン）＝本番の幹。** ここに `push`（プッシュ）すると `tokyotc.com` に公開される。
- **ブランチ＝作業用の枝。** 幹から分かれた別レーン。ここで作業すれば、他の Claude Code が `main` で何をしても巻き込まれない。完成したら幹に **マージ（合流）** して公開。
- **公開タイミング**：`push` または `マージ` をして初めて本番に出る。それまではローカル（自分のPCの中）だけ。

---

## 2. 作業の大きさで使い分ける

| 作業の種類 | どうする | 言い方 |
|---|---|---|
| 大きい作業（図鑑づくり、レポート量産、デザイン変更） | **別ブランチ**で | 「日本酒の会用のブランチを作って、そこで作業して」 |
| 小さい修正（画像差し替え、文言直し） | そのまま `main` で | 「○○を直して。直ったら push して」 |

**複数の Claude Code を同時に動かしているときは、大きい作業はブランチ推奨**（巻き戻し事故を防げる）。

---

## 3. 別ブランチで進めるときの流れ

1. **始める**：「**日本酒の会用のブランチを作って、そこで作業して**」
   → Claude が専用ブランチ（例 `feature/sake-site`）を作り、以降はそこに commit / push する。`main` は触らない。
2. **作っている間**：ブランチに push しても **まだ `tokyotc.com` には出ない**（完成まで非公開で作れる）。
3. **公開する**：「**確認できたから main にマージして公開して**」
   → Claude が枝の内容を `main` に取り込んで push。ここで本番に反映。

---

## 4. 公開（push）のコントロール

- **すぐ公開したい**：「**push して**」「**直して公開して**」
- **公開前に見たい**：「**push はしないで、build と preview で見せて**」→ ローカルで確認 → OKなら「push して」
- **元に戻せる安全網**：作業はこまめに commit してもらう（「ここまでコミットして」）。

---

## 5. 巻き込まれ事故を防ぐ（今回起きたこと）

- 別の Claude Code が `main` で `git reset` などをすると、こちらの **未 push のコミットが巻き戻される**ことがある（今回は復旧できたが、ヒヤリとした）。
- 防ぐには：**① こまめに push して未 push を短く保つ**／**② 大きい作業はブランチに分ける**。

---

## 6. よく使う頼み方（コピペ用）

```
日本酒の会用のブランチを作って、そこで作業して
build と preview で確認して、push はしないで
確認できたから main にマージして公開して
○○を直して。直ったら push して
push する前に、リモートとぶつからないか確認して
```

## 7. 写真を渡すとき

- 元の写真を **`~/Pictures/日本酒の会/`** に入れて、「写真を入れたから○回のレポートに使って」と伝える。
- **ファイル名は連番のままでOK**（Claude が写真を見て酒名・回を判別する）。
- 撮影日（EXIF）で回が自動で分かるので、フォルダ分けは不要。
